Now update the pod to also make use of the NET_ADMIN capability.


Note:
Only make the necessary changes. Do not modify the name of the pod.

Set the capabilities within the container, not at the pod level, as setting them at the pod level is not supported.

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: ubuntu-sleeper
  namespace: default
spec:
  containers:
  - command:
    - sleep
    - "4800"
    securityContext:
      runAsUser: 0
      capabilities:
```

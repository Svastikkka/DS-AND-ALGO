The reason the application is failed is because we have not created the secrets yet. Create a new secret named `db-secret` with the data given below.


- DB_Host = sql01
- DB_User = root
- DB_Password = password123


```bash
kubectl create secret generic db-secret --from-env-file=.env
```

Configure webapp-pod to load environment variables from the newly created secret.


Note: If the pod already exists, you must delete and recreate it. Updating a pod's environment variables (like envFrom) is not allowed on a running pod.

```bash
kubectl explain pod.spec.containers.env.valueFrom.secretKeyRef
```

```bash
DB_Host=sql01
DB_User=root
DB_Password=password123
```

```yaml
apiVersion: v1
kind: Pod
metadata:
  creationTimestamp: "2025-08-17T20:16:18Z"
  generation: 1
  labels:
    name: webapp-pod
  name: webapp-pod
  namespace: default
  resourceVersion: "1087"
  uid: 3d0c0602-20e0-45de-8a63-eecd87301ddd
spec:
  containers:
  - image: kodekloud/simple-webapp-mysql
    imagePullPolicy: Always
    name: webapp
    envFrom:
    - secretRef:
        name: db-secret
    resources: {}
```

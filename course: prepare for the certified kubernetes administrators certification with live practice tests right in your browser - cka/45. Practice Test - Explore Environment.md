From the `ip addr`output shared above, we can see the different interfaces:

`eth0` → The node’s primary interface (has 192.168.60.180/32) → used for external + cluster node-to-node communication.

`flannel.1` → Overlay interface created by Flannel CNI for pod-to-pod routing across nodes (172.17.0.0/32).

`cni0` → The CNI bridge for local pod networking (172.17.0.1/24).
